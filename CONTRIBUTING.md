# Contributing to RollScript

Thank you for your interest in contributing to RollScript!

RollScript is an open-source project developed and maintained by MZ Software GmbH. Contributions from the community are welcome and appreciated.

There are several ways to contribute:

* Reporting bugs
* Suggesting features and improvements
* Improving documentation
* Contributing code
* Creating plugins
* Helping other users in Discussions

## Before You Start

Before opening an issue or submitting a pull request, please check whether a similar issue or pull request already exists.

For general questions, ideas, and discussions, please use GitHub Discussions.

For reproducible bugs and concrete development tasks, please use GitHub Issues.

## Reporting Bugs

Please use the Bug Report issue template whenever possible.

A good bug report should include:

* The RollScript version
* The operating system and version
* The printer model
* The label or roll configuration, if relevant
* A clear description of the problem
* Steps to reproduce the problem
* The expected behavior
* The actual behavior
* Relevant logs or screenshots, if available

Please do not include confidential or sensitive information in public issues.

## Suggesting Features

Feature suggestions are welcome.

For early-stage ideas and concepts, please use GitHub Discussions first. This allows the community to discuss the idea before it becomes a formal development task.

For well-defined feature requests, please use the Feature Request issue template.

When proposing a feature, please explain:

* What problem the feature would solve
* Why the feature would be useful
* How you envision the feature working
* Possible alternatives or workarounds

## Contributing Code

Code contributions are welcome.

The preferred workflow is:

1. Fork the RollScript repository.
2. Create a separate branch for your changes.
3. Make your changes.
4. Build and test your changes locally.
5. Commit your changes.
6. Push your branch to your fork.
7. Open a Pull Request against the `main` branch.

Please keep pull requests focused on a single feature, bug fix, or clearly related group of changes.

Small, focused pull requests are easier to review and are more likely to be accepted quickly.

## Pull Requests

Every pull request is reviewed before it is merged.

A pull request may be:

* Accepted and merged
* Returned with requested changes
* Closed without being merged

A pull request may not be merged if it:

* Introduces unrelated changes
* Breaks the existing build
* Introduces unnecessary complexity
* Does not fit the project goals
* Contains code that cannot reasonably be maintained
* Does not meet the project's licensing requirements

The goal of code review is to maintain the quality, stability, and long-term maintainability of RollScript. Code review is not intended to discourage contributions.

## Coding Style

Please follow the existing coding style and project structure.

When contributing C++ or Qt code:

* Prefer clear and readable code over clever code.
* Keep classes and functions focused.
* Avoid unnecessary dependencies.
* Follow the existing naming conventions.
* Document non-obvious design decisions.
* Keep changes focused and reviewable.

Additional coding guidelines may be added as the project evolves.

## Plugins

RollScript is designed to support an extensible plugin architecture.

Community plugins are welcome.

Plugin contributions should:

* Use the public RollScript plugin API.
* Follow the documented plugin architecture.
* Avoid unnecessary dependencies.
* Clearly document their purpose and functionality.

Additional plugin documentation will be added as the plugin system evolves.

## Licensing

By contributing to RollScript, you agree that your contribution may be distributed under the license of the RollScript project.

Please ensure that you have the right to submit any code, documentation, or other material that you contribute.

Do not submit code copied from another project unless its license is compatible with the RollScript project and the origin and license are properly documented.

## Questions and Discussions

If you are unsure whether something should be an issue, a discussion, or a pull request, feel free to start a discussion first.

The community is welcome to ask questions, share ideas, discuss potential features, and showcase projects built with RollScript.

Thank you for helping make RollScript better!
