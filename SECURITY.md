# Security Policy

## Supported Versions

Security updates are generally provided for the latest stable release of RollScript.

Development versions may contain known or unknown issues and should not be considered production-ready.

| Version               | Supported   |
| --------------------- | ----------- |
| Latest stable release | Yes         |
| Development versions  | Best effort |

## Reporting a Vulnerability

Please do not report security vulnerabilities through public GitHub Issues or Discussions.

If you believe you have found a security vulnerability in RollScript, please report it privately to:

**[security@mzsoft.de](mailto:security@mzsoft.de)**

Please include as much relevant information as possible, such as:

* A description of the vulnerability.
* Steps to reproduce the issue.
* The affected RollScript version.
* The affected operating system or environment.
* Any proof-of-concept code, if available.

Please do not publicly disclose the vulnerability before the issue has been investigated and an appropriate fix or mitigation has been released.

## What to Expect

We will make reasonable efforts to:

* Acknowledge receipt of your report.
* Investigate the reported issue.
* Keep you informed about the status of the investigation where appropriate.
* Release a fix or mitigation when possible.

The time required to investigate and resolve a vulnerability depends on its severity and complexity.

## Scope

This policy applies to the official RollScript software and code maintained in the official RollScript repositories of MZ Software GmbH.

Third-party plugins, extensions, or software are generally outside the scope of this policy unless explicitly stated otherwise.

Thank you for helping us keep RollScript and its users secure.
